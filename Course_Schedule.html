Applied Single Cell Bioinformatics

Andrea Sonnenholzner, Lars Rønn Olsen & Kristoffer Vitting-Seerup

Welcome

Welcome to the Applied Single Cell Bioinformatics! We are really excited to have you and hope that together we can learn a lot!

Course Overview

This course is structured in two parts:

  • The first part introduces various single-cell technologies and analyses, mainly focusing on scRNA-seq. The techniques and tools introduced are broadly applicable to other single-cell data.
  • The second part involves group-based projects where you will work on real single-cell data.

The course follows a “flipped classroom” approach, where you prepare by reading materials and watching lectures, ensuring more time for hands-on computational exercises during in-class sessions.

Expectations

We expect you to prepare from home using the provided materials. These materials will not be covered during the in-person exercises. It's essential to complete the preparations to fully benefit from the exercises and understand the topics.

This design allows for flexible learning and maximizes interaction with teachers and TAs, fostering a more efficient learning environment.

Examination

The examination consists of three parts:

  • Individual multiple-choice quiz in week 2
  • Group project with a final hand-in
  • Oral group presentation of the project

The final grade is based on the overall performance in these three components.

Programming Environment

Throughout the course, we will use R for data analysis, organization, and visualization. We expect you to be familiar with R and the basic tasks required, such as merging datasets, performing group operations, and using ggplot2 for visualization.

An RStudio server with pre-installed packages and data will be used for all exercises. You will need to log in using your DTU credentials.

Seurat will serve as the primary tool for single-cell analysis, supplemented with other R packages when needed.

This is an applied course, focusing on post-quantification analysis of single-cell data.

Prerequisites

For the entire course we will use R to organize, analyze, and visualize data. This also means we REQUIRE that you are already comfortable with programming in R. You do not have time to learn that during this course! If you do not know how to solve basic data manipulation tasks in R (examples listed below), please contact the teacher.

Examples of data analysis you need to know how to do in R prior to course start (without looking it up online):

  • Merge two datasets based on a shared identifier
  • Perform per-group operations based on a grouping column
  • Perform per-row or per-column operations without using loops
  • Use ggplot2 to visualize data, e.g., a scatterplot where points are colored by category

If you are not comfortable with this, you will need to learn fast! There are hundreds of good sources for learning R, including:

  • R for Data Science
  • Modern Statistics for Modern Biology
  • DataCamp Courses
  • Codecademy R Course

Environment

We will use an RStudio server which already has all R packages and exercise data installed.

Log in with your DTU credentials, and you will need to authenticate twice—once for DTU login and once for the RStudio server. Please ensure your credentials are updated with multi-factor authentication before the course begins.

We suggest bookmarking the RStudio server page as you will be using it frequently throughout the course.

Programs

We will use the Seurat package as the backbone for our single-cell analysis and supplement with additional R packages when needed or where Seurat's functionality is lacking.

Starting Point

This course begins after the data has already been quantified. We chose to focus on more applied and advanced analysis methods rather than the quantification process, which is computationally heavy and straightforward. Our focus is on post-quantification analysis of single-cell data.
